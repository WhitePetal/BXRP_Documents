26/*
27Contacts for feedback:
28- pgautron@nvidia.com (Pascal Gautron)
29- mlefrancois@nvidia.com (Martin-Karl Lefrancois)
30
31The top-level hierarchy is used to store a set of instances represented by
32bottom-level hierarchies in a way suitable for fast intersection at runtime. To
33be built, this data structure requires some scratch space which has to be
34allocated by the application. Similarly, the resulting data structure is stored
35in an application-controlled buffer.
36
37To be used, the application must first add all the instances to be contained in
38the final structure, using AddInstance. After all instances have been added,
39ComputeASBufferSizes will prepare the build, and provide the required sizes for
40the scratch data and the final result. The Build call will finally compute the
41acceleration structure and store it in the result buffer.
42
43Note that the build is enqueued in the command list, meaning that the scratch
44buffer needs to be kept until the command list execution is finished.
45
46
47
48Example:
49
50TopLevelASGenerator topLevelAS;
51topLevelAS.AddInstance(instances1, matrix1, instanceId1, hitGroupIndex1);
52topLevelAS.AddInstance(instances2, matrix2, instanceId2, hitGroupIndex2);
53...
54UINT64 scratchSize, resultSize, instanceDescsSize;
55topLevelAS.ComputeASBufferSizes(GetRTDevice(), true, &scratchSize, &resultSize,
56&instanceDescsSize); AccelerationStructureBuffers buffers; buffers.pScratch =
57nv_helpers_dx12::CreateBuffer(..., scratchSizeInBytes, ...); buffers.pResult =
58nv_helpers_dx12::CreateBuffer(..., resultSizeInBytes, ...);
59buffers.pInstanceDesc = nv_helpers_dx12::CreateBuffer(..., resultSizeInBytes,
60...); topLevelAS.Generate(m_commandList.Get(), rtCmdList,
61m_topLevelAS.pScratch.Get(), m_topLevelAS.pResult.Get(),
62m_topLevelAS.pInstanceDesc.Get(), updateOnly, updateOnly ?
63m_topLevelAS.pResult.Get() : nullptr);
64
65return buffers;
66
67*/
